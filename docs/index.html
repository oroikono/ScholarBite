ScholarBite

Your AI-powered daily arXiv research companion. Fresh papers, summarized and ranked, delivered to Slack or WhatsApp every morning.

Get Started How It Works

What it does

Stop scrolling arXiv. Start reading what matters.

📨

Morning Digest

Every day, ScholarBite scans arXiv for papers matching your research interests and delivers a ranked summary to your Slack or WhatsApp.

🎯

Relevance Scoring

Papers are scored 0–10 based on keyword matches, category overlap, and code availability. You see the most relevant work first.

💬

Chat Follow-ups

Reply to any report to dig deeper. Ask about methodology, compare papers, or find related code repos — the agent remembers context.

💻

Code Discovery

Automatically detects GitHub, GitLab, and HuggingFace links in paper metadata. Know immediately if a paper has reproducible code.

🧠

Learns From You

The agent stores conversations locally and adapts over time. The more you interact, the better the recommendations.

🔒

Privacy First

All memory stays on your machine. Use a local LLM (Ollama) for full privacy — only arXiv queries leave your network.

How it works

Metadata-first architecture. Fast, cheap, accurate.

Fetch

Query arXiv API for papers in your categories from the last 24 hours

Score

Rank by keyword matches, category overlap, and code availability (no LLM needed)

Summarize

Top 15 papers get one-line summaries via your chosen LLM

Deliver

Formatted digest sent to Slack or WhatsApp. Reply to chat.

Talk to your papers

Not just notifications. A research companion.

You: Tell me more about paper #3
ScholarBite: That paper proposes a new RAG pipeline that reduces hallucination by 40% using retrieval verification chains. The key insight is...
 
You: Does it have code?
ScholarBite: Yes — PyTorch implementation at github.com/author/rag-verify. Uses HuggingFace Transformers, tested on NQ and TriviaQA.
 
You: How does it compare to yesterday's paper on CRAG?
ScholarBite: Both tackle RAG hallucination but from different angles. Yesterday's CRAG paper uses corrective retrieval post-hoc, while this one embeds verification into the generation loop...

Up and running in 5 minutes

Clone, configure, launch.

# Install
pip install nanobot-ai

# Clone & configure
git clone https://github.com/oroikono/ScholarBite.git
cd ScholarBite
cp config/config.example.json ~/.nanobot/config.json
cp .env.example .env

# Add your API key and research interests, then:
docker compose up -d

Privacy & Security

Your research stays yours.

Local memory

Conversation history and preferences are stored in local files on your machine. Nothing is uploaded to external servers by ScholarBite.

Slack Socket Mode

Outbound WebSocket connection only. No public URL exposed. Works behind firewalls. Minimal permissions: chat:write, channels:read.

Full local option

Use Ollama for LLM processing. The only external call is to arXiv's public API. Paper analysis never leaves your machine.
